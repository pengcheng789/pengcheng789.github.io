JVM 学习日志——自动内存管理篇（1）： Java 内存区域

By Cai Pengcheng

Published 2017-09-23

Java 语言体系与其他静态编程语言的区别除了在编译时存在差别（Java 源码会先被编译成字节码，以实现一次编写，到处运行的特性），还有另一个区别 —— 内存管理机制。在 C 或 C++ 中，开发人员对内存具有直接管理的权利，他们需要为每一个对象申请所需的空间，又必须在对象弃用后，对内存进行回收，防止内存泄漏或内存溢出。而在 Java 中，这些行为将由 JVM 负责，开发人员不用对内存进行直接操作，这使得内存泄露或内存溢出的问题大为减少。但是，这并不代表问题不会出现，若出现内存泄露或内存溢出的问题时，了解 JVM 的内存机制，对问题进行追踪将不会太过艰难。

1. 内存区域划分


这是学习 JVM 内存机制的相当著名发一张图了，根据《 Java 虚拟机规范》制定的。接下来简要的描述下这张图的信息。

在 JVM 运行时的数据区中，共有两大类的运行时数据区，分别为：

  • 线程共享数据区
  • 线程隔离数据区

如上图所示，线程共享数据区有两个内存区域（方法区和堆），线程隔离数据区则有三个内存区域（程序计数器、虚拟机栈和本地方法栈）。

(1). 方法区(Method Area)

“它用于存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码等数据。”这是周志明在《深入理解 Java 虚拟机》中说的一句话，概括得很好，对于我来说，对这个的内存区域作用感觉一目了然。
