JVM 学习日志——自动内存管理篇（1）： Java 内存区域

发表于 2017-09-23 | | 阅读次数:

Java 语言体系与其他静态编程语言的区别除了在编译时存在差别（Java 源码会先被编译成字节码，以实现一次编写，到处运行的特性），还有另一个区别 —— 内存管理机制。在 C 或 C++ 中，开发人员对内存具有直接管理的权利，他们需要为每一个对象申请所需的空间，又必须在对象弃用后，对内存进行回收，防止内存泄漏或内存溢出。而在 Java 中，这些行为将由 JVM 负责，开发人员不用对内存进行直接操作，这使得内存泄露或内存溢出的问题大为减少。但是，这并不代表问题不会出现，若出现内存泄露或内存溢出的问题时，了解 JVM 的内存机制，对问题进行追踪将不会太过艰难。

内存区域划分


这是学习 JVM 内存机制的相当著名发一张图了，根据《 Java 虚拟机规范》制定的。接下来简要的描述下这张图的信息。

在 JVM 运行时的数据区中，共有两大类的运行时数据区，分别为：

  • 线程共享数据区
  • 线程隔离数据区

如上图所示，线程共享数据区有两个内存区域（方法区和堆），线程隔离数据区则有三个内存区域（程序计数器、虚拟机栈和本地方法栈）。

方法区(Method Area)

“它用于存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码等数据。”

– 周志明《深入理解 JVM 虚拟机》

这句话概括得很好，对于我来说，对这个的内存区域作用感觉一目了然。

堆(Heap)

堆是 Java 内存中的大头，在大部分的 Java 应用中，堆内存是虚拟机管理的内存中最大的一块，在虚拟机启动时创建。它的唯一作用就是存放对象实例，几乎所有的对象实例都会在这里被分配。

程序计数器(Program Counter Register)

程序计数器是线程隔离的内存区域，随着线程的创建而创建，占用的空间较小。它的作用是

“通过改变这个计数器的值来选取下一条需要执行的字节码指令。”

– 周志明《深入理解 JVM 虚拟机》

Java 虚拟机是支持多线程的，在 CPU 层面上看，JVM 的多线程是多条线程按照调度算法轮流使用 CPU 资源的，因此，从另一条线程切换回来之时， JVM 需要知道当前线程已经执行到何处。因为每条线程的执行位置都大不相同，因此，这个内存区域必须是线程隔离的。

虚拟机栈(VM Stack)

虚拟机栈是 Java 方法执行的内存模型。方法在执行的时候回创建一个叫栈帧的数据结构（它包含着方法必要的描述和属性，以后再学习它）。方法从调用到执行完成，对应着栈帧在虚拟机栈里从入栈到出栈。

本地方法栈(Native Stack)

本地方法栈与虚拟机栈的作用一致，只是对象——虚拟机栈的服务对像是 Java 方法，而本地方法栈的服务对象是本地方法。他们之间的另一个区别是，在 《 Java 虚拟机规范》中对本地方法栈中方法的数据结构没有作出类似“栈帧”的要求，虚拟机可以自由实现它。
