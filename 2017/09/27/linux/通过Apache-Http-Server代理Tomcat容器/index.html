通过Apache Http Server代理Tomcat容器

发表于 2017-09-27 | 分类于 Linux | 阅读次数:

Apache Tomcat（以下简称Tomcat）是一个小巧的Servelet容器，有着占用内存小，运行稳定等特点，在Java Web开发者中很受欢迎，在生产环境中也不少见。但从设计目的和功能用途上来看，Tomcat主要是用来充当解析Servlet的容器，对于线上环境高并发的场景中，显得有点捉襟见肘，影响服务器性能。如果使用在这方面长袖善舞的Apache Http server（以下简称Apache）来充当前端服务器，负责代理转发和均衡负载，则是一个不错的方案。

配置Apache

a. 启用 proxy_module 和 proxy_http_module

b. 创建代理虚拟主机并启用它

c. 重新加载Apache

d. 如果加载失败，使用 Apache 提供的命令查询配置文件是否存在错误

配置Tomcat

a. 在指定的<Connector>元素中进行恰当的代理设置，该元素一般配置在Tomcat目录下的conf/server.xml中

b. 重启Tomcat，${CATALINA}是Tomcat目录

完成

完成上面的步骤之后，Apache和Tomcat的代理就完成了。发送到80端口的请求将会被转发至8080端口的Tomcat容器进行解析。这看起来似乎跟使用Tomcat单独作为服务器没有什么差别，甚至还多了一步代理转发。其实，整个工作只是完成了一半，配置代理转发是为均衡负载做准备的。当前服务器只有一个Tomcat容器，但一般线上环境不止一个，因此，Apache作为前端服务器处理并发请求，然后根据均衡算法将请求转发至Tomcat容器，可以解决部分并发场景的问题。
