使用Github Pages和Hexo快速搭建博客网站

发表于 2017-09-27 | 分类于 Git | 阅读次数:

这是一篇面向非计算机相关专业人士的文章。

拥有一个低调奢华有内涵的博客应该是很多年轻人觉得比较酷的一件事（这句话瞎编的，只为找个理由写这篇博文）！以我的经验来看，开发一个合格的博客网站，应该要考虑三个地方——UI，博客业务和运行环境。对于不是计算机专业的人们来说，这三个点的实现都阻碍自己开发一个博客的需求。但是！计算机是简化人类工作的，帮助人们更好地生活的，为了照顾不懂计算机知识的人们搭建出自己的博客，各种博客框架涌现出来，著名的莫过于WordPress。但这些框架搭建出来的应用也需要的运行环境，还要花时间去维护这个运行环境，这无疑增加了一点技术难度。但是（又一个但是）,我今天分享的这个由大佬们做出来的东西，完全不用考虑这些！

概述

Github

Github是一个通过Git进行版本控制的软件源代码托管服务，由GitHub公司（曾称Logical Awesome）的开发者Chris Wanstrath、PJ Hyett和Tom Preston-Werner使用Ruby on Rails编写而成。[1]

Github主要是做代码托管的，一般写代码的都会放几个开源项目上去。这里我们关注的是Github提供的另一个功能——Gubhub Pages。它能够将你托管在某个特定仓库的代码当作站点文件解析，实现跟浏览网站一样的效果。而这个功能是完全免费的，我们不用去担心博客的运行环境问题了，虽然，我们只能做一个静态的博客网站。

Hexo

Hexo是使用Node.js开发的一个强大的博客框架，用于动态生成博客页面。通过Gihub Pages提供的功能，我们有一个静态站点空间，再通过Hexo动态生成页面，这可以简单看成我们拥有一个可管理的博客站点了。

步骤

因此，我们要做的事情是：

  1. 创建Github Pages需要的仓库
  2. 使用Hexo生成页面
  3. 将页面推送到仓库上
  4. 欣赏自己的成果

如何添加文章下面会说，现在先对首页进行个性化配置。Hexo能够让我们对博客的参数进行配置，这些参数包括但不限于博客的标题，作者的信息，站点的语言，以及一些第三方服务，如数据统计服务，评论服务等。

首先，我们依据官方文档进行一些配置。用文件管理器打开我们的项目目录，可以看到有以下文件：

其中_config.yml是配置文件，我们进行各种参数配置基本上都是在这里进行配置的，其它文件的作用可参考官方文档。

打开_config.yml文件，Windows用户可以用自带的记事本程序，类Unix用户可以用Vi程序，这里我用的则是Sublime Text。

这里我们主要对第6行到第11行和第15行进行配置（冒号后面需要一个半角空格）：

  • title: 网站标题
  • subtitle: 网站副标题，可以填上自己的个性签名之类的
  • description: 网站描述，一般做SEO关键字时候用，可以先不管它
  • author: 作者名字，可以填上我们想要给自己用的名字
  • language: 站点语言，用中文的话可以填上zh-CN
  • timezone: 时区，在大陆的话，一般是用北京时间，即Asia/Shanghai
  • url: 网站地址，这里我们可以填上之前那个创建的仓库的名字，即username.github.io(username用自己的用户名替换)
